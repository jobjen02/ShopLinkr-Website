<p>Om de ShopLinkr API te gebruiken heb je een API-token nodig. Met een token authenticeer je je verzoeken en bepaal je welke rechten de integratie krijgt.</p>
<h3>Token aanmaken</h3>
<ol>
<li>
<p>Ga naar <strong>Instellingen &gt; API Tokens</strong>.</p>
</li>
<li>
<p>Klik op <strong>Token aanmaken</strong>.</p>
</li>
<li>
<p>Geef je token een herkenbare naam, bijvoorbeeld "ERP Integratie" of "Webshop Sync".</p>
</li>
<li>
<p>Selecteer de rechten die je token nodig heeft. Rechten zijn gegroepeerd per categorie en je kunt per resource kiezen welke acties het token mag uitvoeren.</p>
</li>
<li>
<p>Klik op <strong>Opslaan</strong>.</p>
</li>
<li>
<p>Kopieer je token direct na het aanmaken. Je kunt de token daarna niet meer inzien.</p>
</li>
</ol>
<h3>Token beheren</h3>
<p>Je kunt bestaande tokens beheren via <strong>Instellingen &gt; API Tokens</strong>.</p>
<ul>
<li>
<p><strong>Rechten bewerken</strong> - Pas de rechten van een bestaand token aan.</p>
</li>
<li>
<p><strong>Vernieuwen</strong> - Genereer een nieuwe tokenwaarde met dezelfde rechten. De oude token stopt direct met werken.</p>
</li>
<li>
<p><strong>Intrekken</strong> - Verwijder het token permanent. Alle applicaties die dit token gebruiken verliezen direct toegang.</p>
</li>
</ul>
<h3>Goed om te weten</h3>
<ul>
<li>
<p>Je token wordt maar een keer getoond, direct na het aanmaken. Sla het veilig op, bijvoorbeeld in een wachtwoordmanager.</p>
</li>
<li>
<p>Geef elk token alleen de rechten die het daadwerkelijk nodig heeft. Zo beperk je de impact als een token onverhoopt uitlekt.</p>
</li>
<li>
<p>Als je vermoedt dat een token is gelekt, trek het dan direct in en maak een nieuw token aan.</p>
</li>
</ul>
