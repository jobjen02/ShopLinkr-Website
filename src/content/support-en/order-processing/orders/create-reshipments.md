<p>You create a reshipment when you need to send products to a customer again. For example, because a package arrived damaged, a product was missing or the customer received the wrong item.</p>
<h3>Creating a reshipment</h3>
<ol>
<li>
<p>Go to <strong>Orders</strong> and open the completed order you want to create a reshipment for.</p>
</li>
<li>
<p>Click the actions menu (the three dots) and select <strong>Create reshipment</strong>.</p>
</li>
<li>
<p>Select the products you want to ship again and the quantity.</p>
</li>
<li>
<p>Confirm the reshipment.</p>
</li>
</ol>
<p>A new order is created with the type <strong>Reshipment</strong>. It appears in your orders overview and you can process it the usual way: picking, packing and shipping.</p>
<h3>What happens to the stock?</h3>
<p>When you create a reshipment, stock is automatically reserved for the products in the reshipment. A reshipment is otherwise handled just like a regular order, so it can also be included in the regular pick lists.</p>
<h3>Recognising a reshipment</h3>
<p>A reshipment is recognisable by the <strong>Reshipment</strong> type in the order list. The reshipment is linked to the original order, so you can always see which order the reshipment belongs to.</p>

<h3>Good to know</h3>
<ul>
<li>
<p>You can only create a reshipment from an order with the status: <strong>Processed</strong>.</p>
</li>
<li>
<p>The reshipment goes through the same processing flow as a normal order.</p>
</li>
<li>
<p>The delivery address is taken from the original order.</p>
</li>
</ul>
